# AI 建站提示词模板

本文件收集可复制的建站提示词，用于需求分析、风格定位、UI 规范、首页设计、前端开发、截图审查和部署上线。

---

## 1. 项目需求梳理模板

```text
你是一名资深产品经理、UI/UX 设计师和前端架构师。

我要创建一个网站项目，但我是小白，请你不要直接写代码，先帮我梳理需求。

网站类型：
【企业官网 / SaaS 官网 / 个人作品集 / 电商网站 / 教育培训网站 / 本地生活服务网站 / 后台管理系统】

项目名称：
【填写项目名称】

目标用户：
【填写目标用户】

网站核心目标：
【展示品牌 / 获取客户线索 / 在线销售 / 注册试用 / 展示作品 / 服务预约】

主要功能：
【首页、产品介绍、价格页、案例展示、博客、联系表单、登录注册、后台管理】

我希望的风格：
【高级简洁 / 科技感 / 年轻活泼 / 专业可信 / 温暖治愈 / 奢华高端】

请你输出：
1. 目标用户分析
2. 用户最关心的问题
3. 网站应该突出哪些核心卖点
4. 推荐的网站页面结构
5. 首页应该包含哪些模块
6. 每个模块的作用
7. 哪些内容不建议放
8. 后续开发建议

要求：
- 用小白能看懂的话解释
- 不要空话套话
- 不要直接写代码
```

## 5. 前端开发模板

```text
你是一名资深前端工程师和 UI/UX 设计师。

请根据已经确定的网站需求、首页结构和 UI 设计规范，实现网站首页。

技术栈要求：
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- lucide-react
- 响应式设计

开发要求：
1. 先检查当前项目结构和依赖
2. 先只实现首页
3. 页面必须适配桌面端、平板和手机端
4. 每个页面模块拆成独立组件
5. 样式统一遵守 UI 规范
6. 不要廉价渐变、过多阴影、花哨动画
7. 页面要有高级感、留白感和清晰层级
8. 修改前先说明要改哪些文件
9. 完成后给我本地运行和预览步骤
```

## 6. 截图审查优化模板

```text
你是一名非常严格的资深 UI/UX 设计总监。

下面是我当前网站页面截图。请你从专业设计角度进行审查。

请重点检查：
1. 页面第一眼是否高级
2. 是否有明显视觉层级
3. 标题是否足够突出
4. 文案是否清晰有吸引力
5. 留白是否舒服
6. 模块之间是否拥挤
7. 字体大小是否合理
8. 颜色是否统一
9. 按钮是否明显
10. 卡片是否有廉价感
11. 图标是否统一
12. 是否像模板站
13. 移动端是否可能有问题
14. 是否符合现代 SaaS / 品牌官网审美

请输出：
1. 总体评分，满分 10 分
2. 当前页面最明显的 5 个问题
3. 每个问题为什么不好
4. 每个问题应该怎么改
5. 优先级最高的修改建议
6. 如果可以直接改代码，请列出具体修改方案

不要只说“整体不错”。如果页面很丑，请直接指出。
```

```text
项目信息：

代码托管：【GitHub / Gitee / 还没有上传】

是否有后端：【有 / 没有 / 不确定】
是否需要数据库：【需要 / 不需要 / 不确定】
是否有环境变量：【有 / 没有 / 不确定】

请帮我：
1. 判断最适合小白的部署方案
2. 检查项目是否能正常构建
3. 告诉我需要执行哪些命令

7. 指导我绑定域名

- 每一步都用小白能看懂的话讲
- 优先选择最简单、最稳定的方案
- 如果可以用 Vercel，就优先推荐 Vercel
```

## 8. 文献检索与综述写作网站模板

```text
你是一名资深产品经理、UI/UX 设计师、前端工程师和学术写作工具设计专家。

我想做一个“文献检索 + 可视化分析 + AI 辅助文献综述写作”的网站。

请先不要直接写代码，先帮我完成需求分析、功能规划和页面设计。

核心需求：
1. 支持输入关键词检索文献。
2. 支持设置文献年限，例如起始年份、结束年份、近 3 年、近 5 年、近 10 年。
3. 将检索到的文献统一汇总到一个可视化页面中。
4. 可视化内容包括年份趋势、关键词分布、来源分布、文献类型统计。
5. 每条文献展示标题、作者、年份、来源、DOI/链接、摘要、关键词。
6. 用户可以选择重点文献加入“综述素材库”。
7. 页面底部加入 AI 综述生成区。
8. AI 综述生成区调用 OpenAI 兼容的第三方接口。
9. 支持填写 OpenAI Base URL、模型名，但 API Key 不能写死在前端，应通过后端环境变量调用。
10. 可以生成研究现状、研究热点、研究不足、发展趋势、完整文献综述草稿。

技术建议：
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- lucide-react
- Recharts 或 ECharts
- API Route 调用 OpenAI 兼容接口

请输出：
1. 产品定位
2. 目标用户
3. 核心使用流程
4. 页面结构
5. UI 设计规范
6. 数据结构设计
7. API 接口设计
8. OpenAI 兼容接口调用方案
9. 学术合规提醒
10. 前端实现步骤

注意：
- 不要编造文献。
- 不要编造 DOI、作者、期刊、年份。
- AI 生成内容只能作为文献综述草稿，必须提醒用户人工核对。
- API Key 不允许暴露在前端。
```
