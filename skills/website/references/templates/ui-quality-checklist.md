# UI 质量审查清单

用于检查网站是否现代、专业、高级，尤其适合首页、Landing Page、SaaS 官网、企业官网和作品集页面。

---

## 1. 第一眼观感

| 检查项 | 合格标准 |
|---|---|
| 是否一眼看出网站做什么 | 首屏标题清楚，副标题能解释价值 |
| 是否专业可信 | 色彩克制、排版稳定、无明显廉价感 |
| 是否有高级感 | 留白充足、层级清晰、元素不过度堆砌 |
| 是否像模板站 | 不应出现无意义大图、空洞文案、随机图标 |

---

## 2. 视觉层级

| 检查项 | 合格标准 |
|---|---|
| 主标题 | 页面最醒目，字号和字重足够突出 |
| 副标题 | 解释主标题，不抢主标题层级 |
| CTA 按钮 | 用户能马上知道下一步点哪里 |
| 模块标题 | 每个模块标题清楚，和正文有层级差 |
| 正文说明 | 不要太长，不要大段堆文字 |

---

## 3. 色彩统一

### 常见问题

- 主色太艳，显得廉价
- 一个页面用了太多强调色
- 背景和文字对比不足
- 渐变太花哨
- 卡片颜色不统一

### 建议标准

- 主色 1 个
- 辅助色 1-2 个
- 中性色体系稳定
- CTA 使用最明确的主色
- 背景色尽量克制

---

## 4. 字体与排版

| 项目 | 建议 |
|---|---|
| 字体 | 中文用系统字体或思源黑体，英文用 Inter / Geist / system-ui |
| 标题 | 字号足够大，行高不要太松也不要太挤 |
| 正文 | 16px 左右，行高 1.6 左右 |
| 辅助文字 | 14px 左右，颜色降低但仍可读 |
| 字重 | 不要到处加粗，只突出重点 |

---

## 5. 间距与留白

### 合格标准

- 页面左右边距一致
- 模块上下间距统一
- 卡片内部留白足够
- 按钮不会贴边
- 移动端不拥挤

### 常见问题

- 模块之间太挤
- 卡片内部文字贴边
- 标题和正文距离不统一
- 首屏内容挤在一起

---

## 6. 按钮质量

| 检查项 | 合格标准 |
|---|---|
| 主按钮 | 明确、醒目、只有一个主动作 |
| 次按钮 | 样式弱于主按钮 |
| 文案 | 使用动词，如“立即咨询”“开始免费试用” |
| 状态 | hover / disabled 状态清楚 |
| 尺寸 | 桌面端和移动端都容易点击 |

---

## 7. 卡片质量

### 高级卡片通常具备

- 轻边框
- 小圆角或中等圆角
- 轻阴影或无阴影
- 内边距统一
- 图标风格统一
- 文案简短

### 避免

- 厚重阴影
- 多层阴影
- 彩色卡片乱用
- 图标大小不一
- 内容密度过高

---

## 8. 图片与图标

| 检查项 | 合格标准 |
|---|---|
| 图片 | 风格统一，比例稳定，不拉伸 |
| 插画 | 不要用过时 3D 素材或廉价素材 |
| 图标 | 同一套图标体系，推荐 lucide-react |
| 装饰元素 | 少量使用，不抢内容 |

---

## 9. 响应式检查

必须检查：

- 手机端导航是否可用
- 首屏标题是否过大
- 按钮是否换行合理
- 卡片是否从多列变单列
- 图片是否溢出
- 横向滚动是否出现
- 底部表单是否好填写

---

## 10. 评分标准

| 分数 | 含义 |
|---|---|
| 9-10 | 专业可上线，细节优秀 |
| 7-8 | 基本专业，需要局部优化 |
| 5-6 | 能看但不高级，需要系统优化 |
| 3-4 | 明显模板感或廉价感，需要重构 UI |
| 1-2 | 结构混乱，不建议上线 |

---
