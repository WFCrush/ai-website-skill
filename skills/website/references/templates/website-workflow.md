# AI 建站完整工作流

本文档用于约束 AI 建站流程，避免一上来写代码导致页面难看、结构混乱、部署困难。

---

## 阶段 1：需求分析

### 目标

先弄清楚网站要解决什么问题、给谁看、希望用户做什么。

### 需要确认的信息

| 信息 | 示例 |
|---|---|
| 网站类型 | 企业官网 / SaaS 官网 / 作品集 / 电商 / 教育培训 / 本地服务 |
| 项目名称 | XX Studio / XX CRM / XX 教育 |
| 目标用户 | 中小企业老板 / 学生 / 设计师 / 跨境卖家 |
| 网站目标 | 获取客户 / 注册试用 / 在线购买 / 展示品牌 |
| 核心功能 | 首页 / 产品页 / 价格页 / 案例 / 联系表单 / 登录注册 |
| 希望风格 | 高级简洁 / 科技感 / 年轻活泼 / 专业可信 |
| 参考网站 | Apple / Linear / Stripe / Notion / Framer |

### 输出结果

- 目标用户分析
- 用户痛点
- 核心卖点
- 页面结构
- 首页模块
- 不建议放的内容
- 后续开发建议

## 阶段 2：视觉风格定位

### 目标

让用户先选风格，而不是让 AI 自由发挥。

### 默认提供三套方案

| 方向 | 特点 | 适合项目 |
|---|---|---|
| 极简高级风 | 留白大、颜色克制、层级清晰 | 品牌官网、作品集、高端服务 |
| 科技 SaaS 风 | 明确 CTA、数据感、模块清晰 | SaaS、AI 工具、B2B 产品 |
| 年轻活泼风 | 色彩更明亮、插画更多、语气轻松 | 教育、消费品、社区产品 |

### 每套方案必须说明

- 主色
- 辅助色
- 背景色
- 字体感觉
- 按钮风格
- 卡片风格
- 布局特点
- 图片/插画风格
- 优点
- 缺点
- 推荐指数

## 阶段 3：UI 设计规范

### 目标

形成可执行的设计规则，后续代码必须遵守。

### 规范内容

#### 1. 色彩

| 类型 | 建议 |
|---|---|
| 主色 | 用于主按钮、重点链接、关键强调 |
| 辅助色 | 用于标签、图标背景、轻强调 |
| 背景色 | 尽量选择接近白色或深色体系中的克制颜色 |
| 文字主色 | 保证可读性，不要太浅 |
| 文字次色 | 用于说明文字 |
| 边框色 | 低对比、轻边框 |

#### 2. 字体

- 中文：优先系统字体、思源黑体、阿里巴巴普惠体
- 英文：Inter、Geist、SF Pro、system-ui
- 标题：突出但不夸张
- 正文：可读性优先
- 行高：保持舒适，不要太挤

#### 3. 间距

- 页面最大宽度：通常 1120px - 1280px
- 模块上下间距：桌面端 80px - 120px
- 卡片内边距：24px - 32px
- 移动端边距：16px - 24px

#### 4. 组件

- 按钮：主次分明，同页面不要超过 2 种主要按钮样式
- 卡片：轻边框、轻阴影或无阴影，不要厚重
- 表单：标签清楚、错误提示明确
- 图标：统一线性图标，推荐 lucide-react

## 阶段 4：首页结构与文案

### 推荐首页结构

1. 顶部导航栏
2. Hero 首屏
3. 用户痛点
4. 解决方案
5. 核心功能/服务
6. 优势说明
7. 案例/评价
8. 使用流程
9. 价格/套餐
10. FAQ
11. 底部 CTA
12. 页脚

### 首页首屏必须回答

- 你是谁？
- 你提供什么？
- 对我有什么用？
- 我下一步该点哪里？

## 阶段 5：前端实现

### 推荐技术栈

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- lucide-react

### 实现顺序

1. 检查项目结构
2. 检查依赖
3. 确认入口文件
4. 创建组件目录
5. 实现首页模块
6. 做响应式适配
7. 本地运行预览
8. 根据截图继续优化

### 代码原则

- 模块化
- 不要重复写样式
- 不要硬堆 div
- 不随意删除用户已有文件
- 图片先占位，但位置和比例要设计好

---

## 阶段 6：截图审查优化

### 审查维度

| 维度 | 检查点 |
|---|---|
| 第一眼 | 是否高级、专业、可信 |
| 层级 | 标题、副标题、正文、按钮是否清晰 |
| 留白 | 模块是否拥挤 |
| 色彩 | 是否统一，是否廉价 |
| 字体 | 字号和行高是否舒适 |
| 卡片 | 是否太重、太乱、太模板 |
| CTA | 用户是否知道该点哪里 |
| 移动端 | 手机端是否可用 |

### 输出格式

```md
## 总体评分

## 最明显的 5 个问题

## 修改建议

## 优先级最高的改动

## 具体代码修改方案
```

## 阶段 7：部署上线

### 小白优先推荐

| 平台 | 适合场景 |
|---|---|
| Vercel | Next.js 项目首选 |
| Netlify | 静态站点、React/Vue 前端 |
| Cloudflare Pages | 静态站点、全球访问速度 |
| Railway / Render | 有后端服务时 |

### 部署前检查

- 本地能否运行
- 能否成功 build
- 是否缺环境变量
- 是否有 API 地址写死
- 是否有图片路径错误
- 是否上传到 GitHub

---

## 核心提醒

不要用一句“帮我做个好看的网站”开始。

正确方式是：

```text
请先帮我完成需求分析、风格定位、UI 规范、首页结构，等我确认后再写代码。
网站风格要现代、简洁、高级、有留白，不要廉价渐变和模板感。
```
